# 골프왕부킹 화면 IA 초안
_정보 구조(IA) 및 주요 화면 구성안_

## 1. IA 설계 원칙
- 사용자는 **희망 라운딩 날짜 입력 → 예약 오픈 일정 확인** 흐름을 즉시 이해해야 한다.
- MVP는 정보 탐색 중심이므로, 복잡한 기능보다 **검색-결과-상세** 흐름을 단순하게 유지한다.
- 운영 효율을 위해 관리자 화면은 초기부터 포함한다.
- 공개 사용자 화면과 관리자 운영 화면은 명확히 분리한다.

---

## 2. 전체 사이트맵

```text
/
├─ /search
├─ /courses/[slug]
├─ /about
├─ /disclaimer
├─ /privacy
├─ /terms
├─ /report
└─ /admin
   ├─ /admin/login
   ├─ /admin/dashboard
   ├─ /admin/courses
   ├─ /admin/courses/new
   ├─ /admin/courses/[id]
   ├─ /admin/policies
   ├─ /admin/reports
   ├─ /admin/sources
   └─ /admin/settings
```

## 3. 공개 사용자 화면 구조

## 3.1 `/` 홈 / 검색 시작 화면
### 목적
- 서비스 소개
- 희망 라운딩 날짜 입력
- 지역 필터 선택
- 검색 진입

### 주요 구성
1. 헤더
   - 로고
   - 서비스 소개 링크
   - 면책/안내 링크

2. 히어로 영역
   - 한 줄 가치 제안
   - 날짜 선택기
   - 지역 선택기
   - 검색 버튼

3. 보조 영역
   - 인기 지역 바로가기
   - 예시 검색
   - 서비스 설명

4. 푸터
   - 면책문구
   - 개인정보처리방침
   - 문의

### 핵심 CTA
- `예약 오픈 일정 확인하기`

---

## 3.2 `/search` 검색 결과 화면
### 목적
- 사용자가 입력한 날짜 기준으로 예약 오픈 일정을 빠르게 비교

### 주요 구성
1. 상단 검색 조건 요약
   - 선택한 플레이 날짜
   - 지역 필터
   - 결과 수
   - 검색 수정 버튼

2. 필터 바
   - 지역 재선택
   - 회원권 필요 여부(후속)
   - 정렬 옵션

3. 결과 리스트 영역
   - 오픈 예정 일시 순 정렬
   - 카드형 리스트

4. 결과 카드 항목
   - 골프장명
   - 지역
   - 예약 오픈 예정 일시
   - 정책 요약
   - 회원권 필요 여부
   - 최근 확인일
   - 버튼: 상세보기 / 예약링크 / 홈페이지 / 지도

5. 상태 안내
   - 계산 불가
   - 검토 필요
   - 이미 오픈됨

### MVP 우선 표시 방식
- 캘린더보다 먼저 **리스트형 결과**를 기본으로 한다.
- 날짜 그룹핑이 필요하면 섹션 헤더로 표시한다.

---

## 3.3 `/courses/[slug]` 골프장 상세 화면
### 목적
- 해당 골프장의 예약 정책과 공식 링크를 자세히 확인

### 주요 구성
1. 상단 기본 정보
   - 골프장명
   - 지역/주소
   - 회원권 필요 여부
   - 최근 확인일

2. 예약 정책 영역
   - 정책 요약
   - 정책 원문 또는 운영자 정리 설명
   - 예시 오픈 일정

3. 링크 영역
   - 공식 홈페이지
   - 예약 페이지
   - 지도 보기
   - 대표전화(있다면)

4. 보조 정보
   - 취소 정책 요약
   - 메모
   - 운영 상태

5. 안내문구
   - 정보 변경 가능성
   - 공식 사이트 최종 확인 필요

## 4. 관리자 화면 구조

## 4.1 `/admin/login`
### 목적
- 관리자 인증 진입

### 구성
- 이메일

- 로그인 버튼

## 4.2 `/admin/dashboard`
### 목적
- 운영 현황 요약

### 주요 위젯
- 총 골프장 수
- 검수 완료 수

- 사용자 제보 수
- 최근 수정 내역

---

## 4.3 `/admin/courses`
### 목적
- 골프장 목록 관리

### 주요 구성
- 검색창
- 필터: 지역 / 상태 / 검수 상태
- 테이블 또는 리스트
- 컬럼:
  - 골프장명
  - 지역
  - 상태
  - 검수 상태
  - 최근 확인일
  - 수정 버튼
- 신규 등록 버튼

---

## 4.4 `/admin/courses/new`
### 목적

1. 기본 정보

### 목적

- 처리 상태 변경

## 7. MVP 와이어프레임 우선순위

2. 검색 결과
3. 골프장 상세
4. 관리자 로그인
5. 관리자 골프장 목록
6. 관리자 골프장 입력/수정

### 2차 우선 화면
7. 제보 화면
8. 관리자 제보 관리
9. 관리자 출처 관리
10. about / disclaimer / privacy / terms

---

## 8. 모바일 고려사항
- 홈 검색 폼은 모바일 1열 구성
- 검색 결과 카드는 세로 스택 구조
- 외부 링크 버튼은 터치 영역 넓게
- 관리자 화면은 데스크탑 우선, 모바일은 최소 대응

---

## 9. 디자인 시스템 방향
- 정보형 서비스답게 **가독성 우선**
- 과한 그래픽보다 리스트/상태/배지 중심
- 핵심 상태 배지 예시:
  - 검수 완료
  - 검토 필요
  - 오늘 오픈
  - 이미 오픈
- 색상은 상태 구분 중심으로 단순하게
